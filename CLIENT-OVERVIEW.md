## 1. Introduction

The AICTiG website is a modern, professionally built platform designed to serve as the public-facing digital home for the African Institute for Cybersecurity & Tele-Informatics Governance. It showcases the organization's mission, research, consultancy services, and — most importantly — provides a comprehensive, searchable repository of policy documents across Africa and beyond.

The website is fast, mobile-friendly, works in 7 languages, and includes a built-in analytics dashboard so you can see exactly how visitors are using the platform.

---

## 2. What the Website Does

At a high level, the website serves four main purposes:

1. **Showcases AICTiG** — Presents the organization's mission, vision, focus areas, and consultancy services to the public.
2. **Hosts a Policy Repository** — A fully searchable, filterable collection of policy documents from African Union bodies, regional organizations, and international institutions.
3. **Publishes Content** — Displays news articles, research publications, and a photo gallery, all managed through a content management system.
4. **Engages Visitors** — Provides a contact form, social media links, and tracks visitor engagement through a privacy-respecting analytics platform.

## 4. The Policy Repository

The Policy Repository is the core feature of the website and has been built with significant attention to usability and organization.

### How Documents Are Organized

Documents in the repository are categorized across multiple dimensions:

- **Document Level** — Whether a document is Continental (Africa-wide), Regional, or International in scope
- **Organization** — The body that issued the document, such as:
  - *Continental:* African Union (AU), AU Commission (AUC), AU Development Agency (AUDA-NEPAD), and others
  - *Regional:* ECOWAS, East African Community, SADC, and other regional economic communities
  - *International:* United Nations, ITU, World Bank, European Union, and others
- **Country** — The nation(s) the document applies to
- **Sector/Theme** — The thematic area (matching the 22 focus areas)
- **Year** — When the document was published
- **Language** — The language the document is written in

### Search & Filtering

Visitors can find documents through:
- **Text search** — Type keywords to search across document titles and descriptions
- **Filter sidebar** — Narrow results using any combination of the categories above
- **Focus area links** — Click any focus area to jump directly to matching documents

### Viewing Options

The repository offers two viewing modes:
- **Grid view** — Documents displayed as cards in a grid, ideal for browsing
- **List view** — Documents displayed in a compact list, ideal for scanning many results quickly

### Document Details

Each document page shows:
- The country flag and full document title
- Organization, level, sector, year, and language information
- A direct download link for the PDF document
- A preview button to view online
- Related policy documents from the same region or topic

## 5. Content Management (Backend)

All content on the website — news articles, publications, policy documents, and gallery images — is managed through **WordPress**, a widely used content management system.

### How It Works

- AICTiG staff log in to a separate admin panel (be.aictig.org) to add, edit, or remove content
- Changes made in WordPress automatically appear on the public website — no coding required
- The website pulls content from WordPress in real time, so updates are reflected immediately
- Images and PDF documents are uploaded and stored through WordPress's built-in media library

### What Can Be Managed

| Content Type | What You Can Do |
|---|---|
| **Policy Documents** | Add new documents, assign categories (country, theme, organization, year, language), upload PDFs |
| **Publications** | Create publications with authors, abstracts, content, and cover images |
| **News Articles** | Write and publish news with images, tags, and featured article status |
| **Gallery Images** | Upload photos with titles, captions, dates, and locations |

This means **you do not need a developer to update content** — anyone with WordPress admin access can manage the website's content.

## 7. Analytics & Visitor Insights

The website includes a fully self-hosted analytics platform called **Matomo**, accessible at [analytics.aictig.org](https://analytics.aictig.org). This gives AICTiG complete ownership of its visitor data — no data is shared with Google or any third party.

### What Is Being Tracked

The analytics system captures the following information:

#### Visitor Information
- Number of visitors (daily, weekly, monthly, yearly)
- Which countries visitors are from
- What devices and browsers they use
- How visitors found the website (search engines, social media, direct links, etc.)
- Which external links visitors click on

#### Page Engagement
- Which pages are visited most often
- How long visitors spend on each page
- The path visitors take through the website (entry page → browsing → exit page)

#### Document Activity
- **Downloads** — Every time a visitor downloads a PDF document, it is recorded with the document name
- **Previews** — Every time a visitor opens a document preview, it is logged

#### Search & Filtering Behavior
- What search terms visitors type in the repository
- How many results each search returns (helps identify gaps in the collection)
- Which filters visitors use most often (country, theme, year, etc.)

#### Website Performance
- Page loading speed metrics (how fast pages appear for visitors)
- Core Web Vitals — industry-standard measures of website quality used by Google

### Accessing the Dashboard

The Matomo analytics dashboard is available at:

> **[analytics.aictig.org](https://analytics.aictig.org)**

Log in with the admin credentials to view all reports. The dashboard provides:
- **Real-time view** — See who is on the website right now
- **Standard reports** — Pre-built reports for visitors, pages, downloads, referrers, and more
- **Date range selection** — View data for any time period
- **Export options** — Download reports as PDF or CSV for presentations and reporting

### Privacy

Matomo is self-hosted on AICTiG's own server, meaning:
- All visitor data stays on your server — nothing is sent to third parties
- No cookies are required for basic tracking
- It is compliant with GDPR, CCPA, and other privacy regulations
- Visitors' IP addresses are anonymized by default

## 8. Search Engine Visibility (SEO)

The website has been optimized to appear in search engine results (Google, Bing, etc.) through a number of measures:

- **Descriptive page titles and descriptions** — Every page has a unique title and summary that search engines display in results
- **Structured data** — The website provides machine-readable information about the organization (name, logo, founding date, area of operation) so search engines can display rich results
- **Sitemap** — A file listing all pages on the website, submitted to search engines so they can find and index every page
- **Multi-language signals** — Search engines are told about all 7 language versions so they can show the right version to visitors based on their language
- **Social media previews** — When the website URL is shared on platforms like Facebook, X (Twitter), or LinkedIn, a professional preview card with the page title, description, and image is displayed automatically
- **Mobile-friendly design** — The website is fully responsive, which is a factor search engines use when ranking results

## 9. Performance & Reliability

The website has been built with speed and reliability as priorities:

### Speed
- **Code splitting** — Only the code needed for the current page is loaded, making initial page loads fast
- **Smart caching** — Content that doesn't change frequently is stored locally in the visitor's browser, so repeat visits are near-instant
- **Optimized images** — Images are loaded only when they scroll into view (lazy loading), reducing initial page weight
- **Compressed assets** — All files sent to the browser are minimized and compressed

### Reliability
- **Error protection** — If any part of the website encounters an issue, the error is contained so the rest of the site continues working normally
- **Loading indicators** — While content is being fetched, smooth skeleton placeholders are shown instead of blank screens
- **Fallback handling** — If content cannot be loaded, clear messages are displayed instead of broken pages

### Mobile Support
- The website is fully responsive — it adapts its layout to work on desktops, tablets, and smartphones
- Navigation converts to a mobile-friendly menu on smaller screens
- The search function works as a full-screen overlay on mobile for easy use
- Touch-friendly buttons and controls throughout
